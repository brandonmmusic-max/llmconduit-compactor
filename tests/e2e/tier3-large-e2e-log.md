# llmconduit context compaction — Tier 3 LARGE (>100K token) e2e log

**Date:** 2026-06-14  
**Gateway:** `llmconduit` (fork `brandonmmusic-max/llmconduit`, branch `feat/context-compaction`) — real HTTP path  
**Companion:** `brandonmmusic-max/llmconduit-compactor` — now with **chunked extraction** for large folds + a `/state` introspection endpoint  
**Upstream model:** MiniMax-M3-NVFP4 on `127.0.0.1:9211` (vLLM/b12x, TP4 RTX PRO 6000)  

This extends the Tier 3 test to **>100K-token inputs** to stress summarization/compaction at scale, on **real, public artifacts**:

- **Case A (chat):** real Kentucky Supreme Court opinions from the public CAP corpus (public record).
- **Case B (tools):** the real `llmconduit` Rust source (public OSS), read via tool calls.

Into each input, **5 unique "needle" facts** are planted at increasing depth (~2/26/50/74/96% of the folded region) and tagged (`ALPHA-7731 … ECHO-6673`). The needles live **only** in the folded region (verified: none appear in the kept tail), so recall of a needle proves it survived compaction. Recall-by-depth measures lost-in-the-middle.

## Verdict

- **Compaction fired on every >100K input:** PASS
- **Needle recall across compaction (literal tag in the model's answer):** **9/10** (Case A 5/5; Case B 4/5)
- **Substance preservation:** of the needles whose literal tag was dropped, the underlying content still survived in the state block (e.g. Case B's `CHARLIE-9015` upstream-cooldown finding: the tag prefix was lost but `cooldown`/`failover`/`upstream_failure`/`skipped` are all present). So functional recall is effectively 10/10; tag recall is 9/10.
- **Tool-pair safety (large tool transcript):** PASS
- **Works for BOTH document prose and tool-calls/code** — the explicit goal of this round.

### How it got here — the first >100K run failed, and why
The initial run (coding-agent-only extractor, single big chunks) scored **chat 2/5, tools 1/5**. Diagnosis from the raw per-chunk output:
1. **Prompt/schema mismatch on prose.** The extractor only looked for coding artifacts (objective/decisions/files/tool-outcomes), so big windows of legal opinion text returned an empty `{}` — it skimmed prose as 'nothing durable'. Chat extracted ~2 ops from 128K.
2. **Cap eviction.** When broadened, needles were extracted but routed to `record_fact`, which the merge layer caps and evicts oldest-first — so needles deep in a 100K doc were dropped.
3. **Rendering gap.** Needles correctly routed to typed ops (decision/task) had their tag in the `id` field, which the state-block renderer didn't print — substance survived, tag was invisible.

### Fixes applied (companion), then re-run → chat 5/5, tools 4/5
- `extractor.py` — **chunked extraction**: the rendered archive is split into `COMPACTOR_FOLD_CHUNK_TOKENS` (8K here) windows, extracted concurrently (`COMPACTOR_EXTRACT_CONCURRENCY`=8); a chunk failing drops only that window. Small folds still take one call (unchanged).
- `extractor.py` — **content-adaptive prompt**: summarize reference material/prose into key facts, and **always** capture explicit instructions/labeled notes, **routed to typed ops** (rule→constraint, choice→decision, action/deadline→task) which are not capped — keeping the tag/id in the text.
- `transcript.py` — the state-block renderer now prints the **id/handle** for decisions and tasks, so tags are visible to the downstream model.
- `merge.py` — bumped the bounded-view caps (facts 100, tool-outcomes 50, artifacts 60) so a large document yields a richer digest; high-priority items stay in uncapped typed ops regardless.
- `server.py` — added `GET /state/{session_id}` so the **raw delta ops** (below) can be reviewed.
